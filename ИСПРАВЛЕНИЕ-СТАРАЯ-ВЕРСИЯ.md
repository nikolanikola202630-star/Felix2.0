# 🔧 Исправление: Открывается старая версия

**Проблема:** Бот открывает старую версию вместо EGOIST ACADEMY

**Причина:** Переменная окружения `MINIAPP_URL` в Vercel ещё не обновлена

---

## ✅ Решение (2 способа)

### Способ 1: Через Vercel Dashboard (рекомендуется)

1. **Откройте Vercel Dashboard:**
   https://vercel.com/dashboard

2. **Выберите проект:**
   `felix2-0`

3. **Перейдите в настройки:**
   Settings → Environment Variables

4. **Найдите переменную:**
   `MINIAPP_URL`

5. **Нажмите Edit и измените на:**
   ```
   https://felix2-0.vercel.app/miniapp/egoist.html
   ```

6. **Сохраните:**
   Save

7. **Redeploy:**
   - Deployments → ... (три точки у последнего деплоя)
   - Redeploy
   - Use existing Build Cache
   - Redeploy

8. **Подождите 2-3 минуты**

---

### Способ 2: Через Vercel CLI

```bash
# Удалить старую переменную
vercel env rm MINIAPP_URL production

# Добавить новую
vercel env add MINIAPP_URL production
# Введите: https://felix2-0.vercel.app/miniapp/egoist.html

# Редеплой
vercel --prod
```

## 🐛 Если всё ещё не работает

### 1. Проверить, что файл задеплоен:

```bash
curl -I https://felix2-0.vercel.app/miniapp/egoist.html
```

Должен вернуть `200 OK`

### 2. Проверить переменную окружения:

```bash
vercel env ls
```

Должно быть:
```
MINIAPP_URL    https://felix2-0.vercel.app/miniapp/egoist.html    Production
```

### 3. Проверить webhook:

```bash
curl "https://api.telegram.org/bot8623255560:AAE7sC-7-eWA5LD-ebATDUh6nGUG0pYm03U/getWebhookInfo"
```

Должно быть:
```json
{
  "url": "https://felix2-0.vercel.app/api/webhook",
  "pending_update_count": 0
}
```

### 4. Принудительный редеплой:

```bash
# Сделать пустой commit
git commit --allow-empty -m "redeploy: force update"
git push

# Или через Vercel CLI
vercel --prod --force
```
