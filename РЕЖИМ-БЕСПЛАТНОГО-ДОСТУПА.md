## 📋 Детали изменений

### API Changes

#### 1. Платёжные endpoints

Все платёжные endpoints теперь возвращают:
```json
{
  "success": false,
  "message": "Payment system temporarily disabled. All courses are free.",
  "free_access": true
}
```

Оригинальный код сохранен в комментариях с пометкой `/* PAYMENT: ...`

#### 2. Проверка доступа к курсам

**Было:**
```javascript
// Проверить покупку
const purchased = await checkPurchase(userId, courseId);
if (purchased) {
  hasAccess = true;
}
```

**Стало:**
```javascript
// FREE ACCESS: All courses available
let hasAccess = true;
let reason = 'free_access_period';
```

#### 3. Мои курсы

**Было:**
```javascript
const purchases = await getUserPurchases(userId);
const myCourses = purchases.map(purchase => ...);
```

**Стало:**
```javascript
// FREE ACCESS: Return all courses
const allCourses = coursesData.courses.map(course => ({
  ...course,
  is_free: true,
  free_access_period: true
}));
```

### Frontend Changes

#### 1. Каталог курсов

**Было:**
```html
<div class="course-price">
  ${course.price > 0 ? course.price + ' ₽' : 'Бесплатно'}
</div>
<button>
  ${course.price === 0 ? 'Начать' : 'Подробнее'}
</button>
```

**Стало:**
```html
<div class="meta-item" style="color: var(--success);">
  <span>🎁</span>
  <span>Бесплатно</span>
</div>
<button>Начать обучение</button>
```

#### 2. Страница курса

Убраны:
- Блок с ценой
- Кнопка "Купить"
- Проверка покупки

Добавлено:
- Прямой доступ ко всем урокам
- Бейдж "Бесплатно"

## 🔄 Как вернуть платные функции

### Шаг 1: Раскомментировать API код

Найти все блоки с пометкой `/* PAYMENT:` и раскомментировать:

```javascript
/* PAYMENT: Original code - uncomment to restore
module.exports = async function handler(req, res) {
  // ... original payment code
}
*/
```

### Шаг 2: Применить обратную миграцию

```sql
-- Вернуть платные курсы
UPDATE courses 
SET is_free = false 
WHERE price > 0;
```

### Шаг 3: Обновить фронтенд

Вернуть отображение цен и кнопок покупки в:
- `miniapp/js/catalog.js`
- `miniapp/js/app.js`
- `miniapp/js/academy.js`

### Шаг 4: Тестирование

1. Проверить создание invoice
2. Протестировать покупку через Telegram Stars
3. Проверить начисление комиссий
4. Убедиться в работе refund
