## 🐛 Проблема:

Бот не мог отправить кнопку "Открыть Академию" из-за ошибки:

```
❌ Telegram API error: {
  ok: false,
  error_code: 400,
  description: "Bad Request: inline keyboard button Web App URL 
  'http://localhost:3000/miniapp/index.html' is invalid: 
  Only HTTPS links are allowed"
}
```

**Причина:** Telegram Web Apps требуют HTTPS URL, а мы использовали HTTP (localhost).

---

## ✅ Решение:

Изменили URL в `bot.js` с локального на production:

**Было:**
```javascript
const MINIAPP_URL = 'http://localhost:3000/miniapp/index.html';
```

**Стало:**
```javascript
const MINIAPP_URL = 'https://felix2-0.vercel.app/miniapp/index.html';
```

## 💡 Важные моменты:

### 1. Локальный сервер (localhost:3000)
- Используется только для тестирования в браузере
- Telegram не может открыть HTTP ссылки
- Полезен для отладки дизайна

### 2. Production (Vercel)
- Используется ботом для Mini App
- HTTPS обязателен для Telegram
- Автоматически обновляется при push в Git

### 3. Тестовые данные
- Хранятся в памяти бота (in-memory)
- Сбрасываются при перезапуске
- Для production нужна настоящая БД
