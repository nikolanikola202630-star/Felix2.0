# 🎯 Финальная настройка - 3 шага

**Время:** 5 минут  
**Цель:** Завершить настройку БД и Vercel

---

## Шаг 1: Исправить БД (2 мин)

### 1.1 Применить SQL

1. Открыть Supabase Dashboard → SQL Editor
2. Скопировать содержимое `database/fix-missing-columns.sql`
3. Вставить и нажать **Run**

### 1.2 Ожидаемый результат

```
NOTICE: Колонка courses.price добавлена
UPDATE 10

Success
Rows: 10 (список курсов с ценами)
```

### 1.3 Проверить

```bash
node scripts/analyze-database.js
```

Должно показать:
```
✅ courses: все критические колонки присутствуют
✅ purchases: все критические колонки присутствуют
```

---

## Шаг 2: Добавить переменные в Vercel (2 мин)

### Вариант A: Автоматически (рекомендуется)

```bash
node scripts/setup-vercel-env.js
```

Скрипт автоматически:
- Проверит все переменные в .env
- Добавит их в Vercel Production
- Покажет результат

### Вариант B: Вручную

```bash
# SUPABASE_URL
vercel env add SUPABASE_URL production
# Вставить: https://nnegvsxhspdnsvjpcscn.supabase.co

# SUPABASE_KEY
vercel env add SUPABASE_KEY production
# Вставить: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# И так далее для всех переменных
```

### Проверить переменные

```bash
vercel env ls
```

Должны быть:
- SUPABASE_URL
- SUPABASE_KEY
- TELEGRAM_BOT_TOKEN
- GROQ_API_KEY
- ADMIN_ID
- MINIAPP_URL
- AI_DAILY_LIMIT
- AI_HOURLY_LIMIT
- NODE_ENV

---

## Шаг 3: Деплой и тест (1 мин)

### 3.1 Деплой

```bash
vercel --prod
```

### 3.2 Проверить health check

```bash
curl https://felix2-0.vercel.app/api/health/database
```

Ожидаемый ответ:
```json
{
  "success": true,
  "database": {
    "connected": true,
    "message": "Database connection OK"
  },
  "status": "healthy"
}
```

### 3.3 Проверить API курсов

```bash
curl https://felix2-0.vercel.app/api/courses-full
```

Должен вернуть список курсов с ценами.
