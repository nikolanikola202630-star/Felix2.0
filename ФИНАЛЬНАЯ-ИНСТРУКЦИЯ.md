## 🚀 ЧТО ДЕЛАТЬ СЕЙЧАС (5 минут):

### Шаг 1: Авторизоваться в Vercel

Vercel CLI ждет авторизации. Выполните:

```powershell
vercel login
```

**Откроется браузер с страницей авторизации:**
1. Подтвердить авторизацию
2. Вернуться в PowerShell
3. Дождаться сообщения "Success!"

### Шаг 2: Деплой

После авторизации:

```powershell
vercel --prod
```

**Vercel спросит:**
- Set up and deploy? → **Yes**
- Which scope? → Выбрать ваш аккаунт
- Link to existing project? → **Yes**
- Project name? → **felix2-0** (или выбрать из списка)

**Затем начнется деплой:**
```
Building...
Deploying...
✅ Production: https://felix2-0.vercel.app
```

## 🔧 Настройка после первого деплоя:

### 1. Environment Variables (обязательно!)

Vercel Dashboard → Settings → Environment Variables

**Добавить новые в v7.1:**
```env
SENTRY_DSN=your_sentry_dsn
AI_DAILY_LIMIT=50
AI_HOURLY_LIMIT=10
KV_URL=your_kv_url
KV_REST_API_URL=your_kv_api_url
KV_REST_API_TOKEN=your_kv_token
KV_REST_API_READ_ONLY_TOKEN=your_kv_readonly_token
```

### 2. Создать Vercel KV

1. Vercel Dashboard → Storage
2. Create Database → KV (Redis)
3. Скопировать все переменные
4. Добавить в Environment Variables

### 3. Создать Sentry проект

1. https://sentry.io
2. Создать проект: Felix Bot
3. Скопировать DSN
4. Добавить SENTRY_DSN в Vercel

### 4. Redeploy

После добавления переменных:
```powershell
vercel --prod --force
```

Или через Dashboard:
- Deployments → последний → ⋯ → Redeploy
