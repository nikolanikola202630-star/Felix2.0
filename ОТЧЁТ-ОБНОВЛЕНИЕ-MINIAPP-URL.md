## 📋 Что осталось сделать

### Шаг 1: Обновить переменную в Vercel

**Через Dashboard:**
1. Откройте https://vercel.com/dashboard
2. Выберите проект `felix2-0`
3. Settings → Environment Variables
4. Найдите `MINIAPP_URL`
5. Измените на: `https://felix2-0.vercel.app/miniapp/app.html`
6. Сохраните

**Через CLI:**
```bash
vercel env rm MINIAPP_URL production
vercel env add MINIAPP_URL production
# Введите: https://felix2-0.vercel.app/miniapp/app.html
```

### Шаг 2: Redeploy

```bash
# Commit изменений
git add .
git commit -m "fix: обновить MINIAPP_URL на app.html"
git push

# Или принудительный redeploy
vercel --prod
```

## 🐛 Troubleshooting

### Проблема: Бот все еще открывает старую версию

**Причины:**
1. Telegram кэширует URL мини-приложения
2. Переменная окружения не обновлена в Vercel
3. Не выполнен redeploy

**Решения:**

**1. Очистить кэш Telegram:**
- Android: Настройки → Данные и память → Очистить кэш
- iOS: Настройки → Данные и память → Очистить кэш
- Desktop: Настройки → Продвинутые → Очистить кэш

**2. Использовать Telegram Web:**
- Откройте https://web.telegram.org
- Найдите бота @fel12x_bot
- Попробуйте открыть приложение

**3. Подождать:**
- Telegram обновляет кэш автоматически через 5-10 минут
- Просто подождите и попробуйте снова

**4. Проверить Vercel:**
```bash
# Проверить переменные
vercel env ls

# Должно быть:
# MINIAPP_URL    https://felix2-0.vercel.app/miniapp/app.html    Production

# Если нет, обновить:
vercel env rm MINIAPP_URL production
vercel env add MINIAPP_URL production
vercel --prod
```

### Проблема: 404 Not Found

**Причина:** Файл app.html не задеплоен

**Решение:**
```bash
# Проверить, что файл существует
ls miniapp/app.html

# Если есть, redeploy
vercel --prod

# Проверить
curl https://felix2-0.vercel.app/miniapp/app.html
```

### Проблема: Стили не применяются

**Причина:** CSS файлы не загружаются

**Решение:**
```bash
# Проверить файлы
ls miniapp/styles/brandbook-variables.css
ls miniapp/css/brandbook-components.css

# Проверить в браузере (F12 → Network)
# Должны загрузиться без ошибок 404

# Если 404, redeploy
vercel --prod
```
