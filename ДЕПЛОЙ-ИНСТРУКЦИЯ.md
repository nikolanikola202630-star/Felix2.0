### 1. Установите переменные окружения в Vercel

```bash
vercel env add BOT_TOKEN

vercel env add REFERRAL_BOT_TOKEN
# Введите: 8609120719:AAHsLIpWnc3i7MwcEsmfkNTeFIucZqukx9g

vercel env add GROQ_API_KEY
# Введите: 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

vercel env add DATABASE_URL
# Введите: postgresql://postgres:zOEcmylUNd12YCKg@db.nnegvsxhspdnsvjpcscn.supabase.co:5432/postgres

vercel env add ADMIN_ID
# Введите: 8264612178
```

**Важно:** Для каждой переменной выберите окружения:
- Production: Yes
- Preview: Yes  
- Development: Yes

### 2. Выполните деплой

```bash
# Добавьте изменения в git
git add .
git commit -m "feat: database migrations applied, ready for production"

# Отправьте на GitHub (автоматически задеплоится на Vercel)
git push origin main
```

### 3. Установите webhook'и для ботов

После успешного деплоя (когда получите URL от Vercel):

```bash
# Запустите скрипт установки webhook'ов
node scripts/sync-bots.js
```

Скрипт автоматически:
- Установит webhook для основного бота (@fel12x_bot)
- Установит webhook для реферального бота (@ref_bot)
- Проверит статус обоих ботов

## 🔍 Проверка после деплоя

## 📱 Обновите MINIAPP_URL

После деплоя обновите URL в переменных окружения:

```bash
vercel env add MINIAPP_URL
# Введите: https://your-actual-project.vercel.app/miniapp/elite.html
```

Замените `your-actual-project` на реальный URL вашего проекта.

## ⚠️ Возможные проблемы

### Проблема: "Environment variable not found"
**Решение:** Убедитесь, что все переменные добавлены для всех окружений (Production, Preview, Development)

### Проблема: "Webhook setup failed"
**Решение:** 
1. Проверьте, что деплой завершился успешно
2. Убедитесь, что URL доступен
3. Запустите `node scripts/sync-bots.js` повторно

### Проблема: "Database connection failed"
**Решение:**
1. Проверьте DATABASE_URL в Vercel
2. Убедитесь, что пароль правильный
3. Проверьте доступность Supabase
