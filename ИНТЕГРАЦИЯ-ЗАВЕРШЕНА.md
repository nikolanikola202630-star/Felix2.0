### Шаг 2: Проверить API endpoints

**Тест rate limiting:**
```bash
# Отправить 101 запрос подряд - должен вернуть 429
for i in {1..101}; do
  curl https://felix2-0.vercel.app/api/courses-full
done
```

**Тест валидации:**
```bash
# Отправить невалидные данные
curl -X POST https://felix2-0.vercel.app/api/admin-courses \
  -H "Content-Type: application/json" \
  -d '{"action":"addCourse","userId":"invalid"}'
# Должен вернуть 400 Bad Request
```

**Тест создания invoice:**
```bash
curl -X POST https://felix2-0.vercel.app/api/payments/create-invoice \
  -H "Content-Type: application/json" \
  -d '{"course_id":1,"user_id":8264612178}'
# Должен вернуть invoice_link
```

## 🎓 Обучение команды

### Для разработчиков:

**Как добавить rate limiting в новый endpoint:**
```javascript
const { apiLimiter } = require('../lib/middleware/rate-limit');

module.exports = async (req, res) => {
  apiLimiter(req, res, async () => {
    // ваш код
  });
};
```

**Как добавить валидацию:**
```javascript
const { validate, userSchema } = require('../lib/middleware/validate');

module.exports = async (req, res) => {
  validate(userSchema)(req, res, async () => {
    // ваш код
  });
};
```

**Как добавить Telegram auth:**
```javascript
const { telegramAuth } = require('../lib/middleware/telegram-auth');

module.exports = async (req, res) => {
  telegramAuth(req, res, async () => {
    // req.telegramUser содержит данные пользователя
  });
};
```
