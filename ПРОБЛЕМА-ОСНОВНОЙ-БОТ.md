# ❌ Проблема: Основной бот не работает

## Диагностика

### ✅ Webhook endpoint работает:
```bash
curl https://felix2-0.vercel.app/api/webhook
```
Ответ: `{"status":"ok","bot":"Felix Academy - EGOIST ECOSYSTEM","version":"v9.0 FULL"}`

### ❓ Возможные причины:

1. **Webhook не установлен для основного бота**
   - Нужен `TELEGRAM_BOT_TOKEN` основного бота
   - Webhook должен указывать на `https://felix2-0.vercel.app/api/webhook`

2. **Переменные окружения не установлены в Vercel**
   - `TELEGRAM_BOT_TOKEN` - токен основного бота
   - `GROQ_API_KEY` - ключ Groq AI
   - `DATABASE_URL` - строка подключения PostgreSQL
   - `SUPABASE_URL` - URL Supabase
   - `SUPABASE_KEY` - ключ Supabase

3. **База данных не подключена**
   - Ошибка SSL сертификата
   - Нужно настроить переменные БД

## 🔧 Решение

### Шаг 1: Установить переменные окружения в Vercel

```bash
# Перейти в Vercel Dashboard
https://vercel.com/nikolanikola202630-stars-projects/felix2-0/settings/environment-variables

# Добавить переменные:
TELEGRAM_BOT_TOKEN=<токен_основного_бота>
GROQ_API_KEY=<ключ_groq>
DATABASE_URL=<postgresql_connection_string>
SUPABASE_URL=<supabase_url>
SUPABASE_KEY=<supabase_anon_key>
```

### Шаг 2: Установить webhook для основного бота

После установки `TELEGRAM_BOT_TOKEN` в Vercel:

```bash
# Локально (если есть токен в .env):
node scripts/setup-webhook-final.js

# Или через API напрямую:
curl "https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://felix2-0.vercel.app/api/webhook"
```

### Шаг 3: Проверить webhook

```bash
curl "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"
```

Должен вернуть:
```json
{
  "ok": true,
  "result": {
    "url": "https://felix2-0.vercel.app/api/webhook",
    "has_custom_certificate": false,
    "pending_update_count": 0
  }
}
```

### Шаг 4: Протестировать бота

Отправить `/start` в бота - должен ответить приветствием с EGOIST брендингом.

## 🚀 Быстрое решение

Если у тебя есть токен основного бота, выполни:

```bash
# 1. Установи переменные в Vercel через UI или CLI:
vercel env add TELEGRAM_BOT_TOKEN
vercel env add GROQ_API_KEY
vercel env add DATABASE_URL
vercel env add SUPABASE_URL
vercel env add SUPABASE_KEY

# 2. Redeploy:
vercel --prod

# 3. Установи webhook:
curl "https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://felix2-0.vercel.app/api/webhook"

# 4. Проверь:
curl "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"
```

## 💡 Альтернатива: Работа без БД

Если БД пока не нужна, можно временно отключить DB calls:

В `api/webhook-handler.js` обернуть все DB вызовы в try-catch:

```javascript
try {
  await db.getOrCreateUser(...);
} catch (error) {
  console.log('DB not available, skipping');
}
```

Бот будет работать, но без сохранения истории и статистики.
