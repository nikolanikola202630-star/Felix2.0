## 🚀 Быстрый старт

### 1. Запустить сервер

```powershell
node server-local.js
```

**Вывод:**
```
🚀 Felix Bot v9.0 - Local Server Started!

📍 Server running at: http://localhost:3000
📍 Webhook endpoint: http://localhost:3000/api/webhook

🧪 Test commands:
   Health check: curl http://localhost:3000/api/webhook
   Or: Invoke-RestMethod -Uri "http://localhost:3000/api/webhook"
```

```powershell
Invoke-RestMethod -Uri "http://localhost:3000/api/webhook"
```

## 🌐 Подключение Telegram (через ngrok)

### Шаг 1: Установить ngrok

**Windows:**
```powershell
# Скачать с https://ngrok.com/download
# Или через Chocolatey:
choco install ngrok
```

**Или скачать напрямую:**
https://ngrok.com/download

### Шаг 2: Запустить ngrok

```powershell
ngrok http 3000
```

**Вывод:**
```
ngrok

Session Status                online
Account                       Your Account
Version                       3.x.x
Region                        Europe (eu)
Latency                       -
Web Interface                 http://127.0.0.1:4040
Forwarding                    https://abc123.ngrok-free.app -> http://localhost:3000

Connections                   ttl     opn     rt1     rt5     p50     p90
                              0       0       0.00    0.00    0.00    0.00
```

**Скопируй HTTPS URL:** `https://abc123.ngrok-free.app`

### Шаг 3: Установить webhook

```powershell
$token = "8623255560:AAE7sC-7-eWA5LD-ebATDUh6nGUG0pYm03U"
$ngrokUrl = "https://abc123.ngrok-free.app"  # Замени на свой URL из ngrok

$body = @{
    url = "$ngrokUrl/api/webhook"
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://api.telegram.org/bot$token/setWebhook" `
  -Method Post `
  -Body $body `
  -ContentType "application/json"
```

**Ожидаемый ответ:**
```json
{
  "ok": true,
  "result": true,
  "description": "Webhook was set"
}
```

### Шаг 4: Проверить webhook

```powershell
Invoke-RestMethod -Uri "https://api.telegram.org/bot$token/getWebhookInfo"
```

**Ожидаемый ответ:**
```json
{
  "ok": true,
  "result": {
    "url": "https://abc123.ngrok-free.app/api/webhook",
    "has_custom_certificate": false,
    "pending_update_count": 0
  }
}
```

## 🔧 Настройки

### Изменить порт

```powershell
$env:PORT=8080
node server-local.js
```

Или в коде `server-local.js`:
```javascript
const PORT = 8080; // Вместо 3000
```

### Без ngrok (только локальное тестирование)

Можно тестировать через curl/Invoke-RestMethod без Telegram:

```powershell
# Симуляция Telegram webhook
$body = @{
    message = @{
        chat = @{ id = 123456 }
        from = @{ id = 123456; first_name = "Test" }
        text = "/start"
    }
} | ConvertTo-Json -Depth 10

Invoke-RestMethod -Uri "http://localhost:3000/api/webhook" `
  -Method Post `
  -Body $body `
  -ContentType "application/json"
```

## 🐛 Troubleshooting

### Ошибка: "Cannot find module"

```powershell
npm install
```

### Ошибка: "Port already in use"

```powershell
# Найти процесс на порту 3000
netstat -ano | findstr :3000

# Убить процесс (замени PID)
taskkill /PID <PID> /F

# Или используй другой порт
$env:PORT=8080
node server-local.js
```

### Ошибка: "ECONNREFUSED" в Groq API

Проверь интернет соединение и API ключ в `api/webhook-test.js`

### ngrok показывает "ERR_NGROK_108"

```powershell
# Перезапусти ngrok
# Ctrl+C
ngrok http 3000
```

### Telegram не отвечает

1. Проверь webhook:
```powershell
Invoke-RestMethod -Uri "https://api.telegram.org/bot$token/getWebhookInfo"
```

2. Проверь ngrok работает:
```powershell
Invoke-RestMethod -Uri "https://abc123.ngrok-free.app/api/webhook"
```

3. Проверь логи в консоли сервера
