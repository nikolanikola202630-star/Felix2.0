Felix Bot v7.1 оснащен полной автономной системой для автоматизации разработки, деплоя и мониторинга.

## 🔄 Автоматизация Git

### Скрипт auto-commit.js

**Расположение:** `scripts/auto-commit.js`

**Команды:**

```bash
# Автоматические коммиты каждые 5 минут
node scripts/auto-commit.js start

# Разовый коммит с анализом изменений
node scripts/auto-commit.js commit

# Открыть GitHub Desktop
node scripts/auto-commit.js open

# Создать Pull Request
node scripts/auto-commit.js pr

# Синхронизация с remote
node scripts/auto-commit.js sync

# Умный коммит (анализ + группировка)
node scripts/auto-commit.js smart
```

**Возможности:**
- ✅ Автоматический анализ изменений
- ✅ Генерация commit messages
- ✅ Группировка изменений по типам
- ✅ Интеграция с GitHub Desktop
- ✅ Создание PR
- ✅ Синхронизация веток

## 🚀 CI/CD Pipeline

### GitHub Actions Workflows

#### 1. ci-cd-full.yml
**Триггеры:** push, pull_request на main

**Jobs:**
1. **test** - Запуск тестов
2. **security** - Проверка безопасности
3. **database** - Проверка миграций
4. **backup** - Backup БД
5. **deploy** - Деплой на Vercel
6. **migrate** - Применение миграций
7. **smoke-tests** - Smoke тесты
8. **cache-warmup** - Прогрев кэша
9. **notify** - Уведомления

#### 2. backup.yml
**Триггер:** Ежедневно в 03:00 UTC

**Действия:**
- Backup PostgreSQL
- Backup Redis
- Сохранение в GitHub Artifacts
- Уведомление в Telegram

#### 3. test.yml
**Триггер:** push, pull_request

**Действия:**
- Unit тесты
- Integration тесты
- Coverage report
- Загрузка в Codecov

#### 4. auto-update.yml
**Триггер:** Еженедельно в понедельник 09:00 UTC

**Действия:**
- Обновление зависимостей
- Запуск тестов
- Создание PR

## 🔄 Автоматический деплой

### Vercel Integration

**Триггеры:**
- Push в main → Production deploy
- Push в dev → Preview deploy
- Pull Request → Preview deploy

**Процесс:**
1. GitHub Actions запускает тесты
2. При успехе → Vercel деплоит
3. Применяются миграции
4. Прогревается кэш
5. Smoke тесты
6. Уведомление

**Rollback:**
- Автоматический при ошибке
- Ручной через Vercel Dashboard
- Через GitHub Actions (revert commit)

## 🔧 Graceful Shutdown

### Автоматическая обработка

**Модуль:** `lib/shutdown.js`

**Обработка:**
- SIGTERM, SIGINT
- uncaughtException
- unhandledRejection

**Действия:**
1. Прекращение приема новых запросов
2. Завершение текущих запросов (30s timeout)
3. Закрытие соединений БД
4. Закрытие Redis соединений
5. Flush логов в Sentry
6. Graceful exit

## 📦 Backup стратегия

### Автоматические backup

**Расписание:**
- Ежедневно в 03:00 UTC
- Перед каждым деплоем
- Перед миграциями

**Что сохраняется:**
- PostgreSQL dump
- Redis snapshot
- Environment variables (encrypted)
- Конфигурации

**Хранение:**
- GitHub Artifacts (90 дней)
- S3 (опционально, бессрочно)

**Восстановление:**
```bash
# Скачать backup
gh run download <run-id>

# Восстановить БД
psql $DATABASE_URL < backup.sql

# Восстановить Redis
redis-cli --rdb backup.rdb
```
