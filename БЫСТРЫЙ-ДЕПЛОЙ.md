## 🚀 Шаги деплоя (выполнять по порядку!)

### Шаг 1: Установите переменные окружения в Vercel

Выполните команды и вводите значения из `.env.local`:

```bash
vercel env add BOT_TOKEN production
# Введите: 8623255560:AAE7sC-7-eWA5LD-ebATDUh6nGUG0pYm03U

vercel env add REFERRAL_BOT_TOKEN production
# Введите: 8609120719:AAHsLIpWnc3i7MwcEsmfkNTeFIucZqukx9g

vercel env add GROQ_API_KEY production
# Введите: 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

vercel env add DATABASE_URL production
# Введите: postgresql://postgres:zOEcmylUNd12YCKg@db.nnegvsxhspdnsvjpcscn.supabase.co:5432/postgres

vercel env add ADMIN_ID production
# Введите: 8264612178
```

### Шаг 2: Закоммитьте изменения

```bash
git add .
git commit -m "feat: database migrations applied, production ready"
git push origin main
```

Vercel автоматически начнет деплой. Дождитесь завершения (2-3 минуты).

### Шаг 3: Получите URL проекта

После деплоя:
1. Откройте Vercel Dashboard
2. Скопируйте Production URL (например: `https://felix-academy.vercel.app`)

### Шаг 4: Установите webhook'и

Замените `YOUR_VERCEL_URL` на ваш реальный URL:

```bash
# Установите переменную окружения для скрипта
$env:WEBHOOK_URL="https://YOUR_VERCEL_URL"

# Запустите скрипт установки webhook'ов
node scripts/sync-bots.js
```

Скрипт установит webhook'и для обоих ботов и покажет статистику.

### Шаг 5: Обновите MINIAPP_URL

```bash
vercel env add MINIAPP_URL production
# Введите: https://YOUR_VERCEL_URL/miniapp/elite.html
```

Затем сделайте редеплой:
```bash
vercel --prod
```
