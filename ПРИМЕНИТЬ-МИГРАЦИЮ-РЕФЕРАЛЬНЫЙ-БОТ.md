## Как применить

### Вариант 1: Через Supabase Dashboard (РЕКОМЕНДУЕТСЯ)

1. Открой https://supabase.com/dashboard
2. Выбери проект Felix Academy
3. Перейди в **SQL Editor**
4. Скопируй содержимое файла `006-partner-referral-customization-simple.sql`
5. Вставь в редактор
6. Нажми **RUN** (или CTRL+Enter)

### Вариант 2: Через psql

```bash
psql "postgresql://postgres.xxx:password@aws-0-eu-central-1.pooler.supabase.com:6543/postgres" -f database/migrations/006-partner-referral-customization-simple.sql
```

## Что создается

### 6 новых таблиц:

1. **partner_referral_settings** - Настройки бота партнера
2. **partner_access_log** - Лог выполнения условий
3. **partner_quiz_questions** - Вопросы квиза
4. **partner_quiz_results** - Результаты квиза
5. **partner_form_fields** - Поля анкеты
6. **partner_form_responses** - Ответы на анкету

### Тестовые данные:

- Настройки для админов (1907288209, 8264612178)
- 3 примера вопросов квиза

## Проверка

После применения выполни:

```sql
-- Проверка таблиц
SELECT COUNT(*) FROM partner_referral_settings;
SELECT COUNT(*) FROM partner_quiz_questions;

-- Проверка настроек админов
SELECT * FROM partner_referral_settings WHERE partner_user_id IN (1907288209, 8264612178);

-- Проверка квиза
SELECT * FROM partner_quiz_questions WHERE partner_user_id = 1907288209;
```

Должно вернуть:
- 2 записи в partner_referral_settings
- 3 записи в partner_quiz_questions

## Если ошибка

### Ошибка: "relation users does not exist"

Сначала примени базовую схему:
```sql
-- Выполни сначала
database/FULL-SCHEMA-COMBINED.sql
```

### Ошибка: "duplicate key value"

Таблицы уже существуют. Можно пропустить или удалить:
```sql
DROP TABLE IF EXISTS partner_form_responses CASCADE;
DROP TABLE IF EXISTS partner_quiz_results CASCADE;
DROP TABLE IF EXISTS partner_form_fields CASCADE;
DROP TABLE IF EXISTS partner_quiz_questions CASCADE;
DROP TABLE IF EXISTS partner_access_log CASCADE;
DROP TABLE IF EXISTS partner_referral_settings CASCADE;
```

Затем применить миграцию заново.
