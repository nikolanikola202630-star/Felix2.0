## 📋 Чек-лист перед деплоем

### Обязательные шаги

#### 1. Переменные окружения ⚠️

Создайте `.env.local` (если ещё не создан):

```env
# Telegram Bot
TELEGRAM_BOT_TOKEN=your_bot_token_here
BOT_TOKEN=your_bot_token_here

# Supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your_supabase_anon_key_here

# Groq AI
GROQ_API_KEY=your_groq_api_key_here

# Admin
ADMIN_ID=your_telegram_id
ADMIN_IDS=your_telegram_id,another_admin_id

# App URLs
MINIAPP_URL=https://your-app.vercel.app/miniapp/app.html
WEB_APP_URL=https://your-app.vercel.app/miniapp/app.html

# Environment
NODE_ENV=production
```

**Где получить:**
- `TELEGRAM_BOT_TOKEN`: [@BotFather](https://t.me/BotFather) → /newbot
- `SUPABASE_URL` и `SUPABASE_KEY`: [supabase.com](https://supabase.com) → Settings → API
- `GROQ_API_KEY`: [console.groq.com](https://console.groq.com) → API Keys
- `ADMIN_ID`: [@userinfobot](https://t.me/userinfobot)

#### 2. База данных ⚠️

**Вариант A: Через Supabase Dashboard**

1. Откройте https://supabase.com
2. Создайте новый проект
3. Перейдите в SQL Editor
4. Выполните `database/FULL-SCHEMA-COMBINED.sql`

**Вариант B: Через скрипт**

```bash
npm run migrate:up
```

#### 3. Проверка локально ✅

## 🚀 Деплой на Vercel

### Шаг 1: Установить Vercel CLI

```bash
npm i -g vercel
```

### Шаг 2: Залогиниться

```bash
vercel login
```

### Шаг 3: Связать проект (первый раз)

```bash
vercel link
```

Выберите:
- Scope: Ваш аккаунт
- Link to existing project? No
- Project name: felix-academy-v12
- Directory: ./

### Шаг 4: Добавить переменные окружения

```bash
# Добавить все переменные из .env.local
vercel env add TELEGRAM_BOT_TOKEN
vercel env add SUPABASE_URL
vercel env add SUPABASE_KEY
vercel env add GROQ_API_KEY
vercel env add ADMIN_ID
vercel env add NODE_ENV
```

Или через Dashboard:
1. Откройте https://vercel.com/dashboard
2. Выберите проект
3. Settings → Environment Variables
4. Добавьте все переменные

### Шаг 5: Деплой

```bash
# Preview деплой (для тестирования)
vercel

# Production деплой
vercel --prod
```

### Шаг 6: Получить URL

После деплоя вы получите URL:
```
https://felix-academy-v12.vercel.app
```

Сохраните его!

## 🤖 Настройка Telegram бота

### Шаг 1: Обновить MINIAPP_URL

В Vercel Dashboard:
1. Settings → Environment Variables
2. Найдите `MINIAPP_URL`
3. Измените на: `https://your-app.vercel.app/miniapp/app.html`
4. Redeploy: Deployments → ... → Redeploy

### Шаг 2: Установить webhook

```bash
curl -X POST "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.vercel.app/api/webhook",
    "allowed_updates": ["message", "callback_query", "inline_query"]
  }'
```

Или используйте скрипт:

```bash
# Создайте scripts/set-webhook-v12.js
node scripts/set-webhook-v12.js
```

### Шаг 3: Проверить webhook

```bash
curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getWebhookInfo"
```

Должно быть:
```json
{
  "ok": true,
  "result": {
    "url": "https://your-app.vercel.app/api/webhook",
    "has_custom_certificate": false,
    "pending_update_count": 0
  }
}
```

## 🐛 Troubleshooting

### Проблема: Бот не отвечает

**Решение:**
1. Проверьте webhook: `curl https://api.telegram.org/bot<TOKEN>/getWebhookInfo`
2. Проверьте логи: `vercel logs`
3. Проверьте переменные: `vercel env ls`
4. Переустановите webhook

### Проблема: Приложение не открывается

**Решение:**
1. Проверьте `MINIAPP_URL` в переменных окружения
2. Проверьте, что файл `miniapp/app.html` существует
3. Проверьте логи браузера (F12)

### Проблема: Курсы не загружаются

**Решение:**
1. Проверьте Supabase: `npm run test:supabase`
2. Проверьте API: `curl https://your-app.vercel.app/api/app?action=getCourses`
3. Проверьте таблицу courses в Supabase Dashboard

### Проблема: Ошибка аутентификации

**Решение:**
1. Проверьте `TELEGRAM_BOT_TOKEN` в переменных
2. Проверьте initData в консоли браузера (F12)
3. Убедитесь, что приложение открыто через Telegram

### Проблема: 404 Not Found

**Решение:**
1. Проверьте `vercel.json`
2. Проверьте `api/_router.js`
3. Redeploy: `vercel --prod`
