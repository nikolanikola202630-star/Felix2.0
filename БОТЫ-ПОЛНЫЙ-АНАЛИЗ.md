### Обработка Сообщений

**Текстовые сообщения:**
1. Проверка на команду (`/`)
2. Если команда - выполнить обработчик
3. Если обычное сообщение - отправить в AI
4. Проверить лимит AI (50/день)
5. Получить ответ от Groq
6. Сохранить в историю
7. Отправить пользователю

**Callback queries (кнопки):**
- Обрабатываются в `lib/bot/callbacks.js`
- Поддержка inline клавиатур
- Навигация по меню

## 🔗 БОТ #2: РЕФЕРАЛЬНЫЙ БОТ (Referral Bot)

### Идентификация
- **Username:** @ref_bot (предположительно)
- **Token:** `8609120719:AAHsLIpWnc3i7MwcEsmfkNTeFIucZqukx9g`
- **Webhook:** `https://felix2-0.vercel.app/api/referral-bot-v2`
- **Файл обработчика:** `api/referral-bot-v2.js`
- **Версия:** v2.0

### Основной функционал

#### 1. Обработка Реферальных Переходов

**Процесс:**
1. Пользователь переходит по ссылке: `https://t.me/ref_bot?start=ref_123456`
2. Реферальный бот получает `/start ref_123456`
3. Извлекает ID партнера (123456)
4. Проверяет партнера в БД
5. Сохраняет клик в `referral_clicks`
6. Проверяет условия доступа
7. Показывает приветствие
8. Перенаправляет в основного бота

#### 2. Трекинг Кликов

**Данные клика:**
- Partner User ID
- Referral Code
- IP Hash (для приватности)
- User Agent
- Session ID
- Is Unique (проверка спама)
- Blocked Reason (если заблокирован)

**Защита от спама:**
- Максимум 5 кликов в час с одного IP
- Хеширование IP (SHA-256)
- Проверка уникальности

#### 3. Условия Доступа (Access Conditions)

**Типы условий:**

**A. Подписка на канал**
```json
{
  "type": "telegram_subscription",
  "channel": "@your_channel"
}
```

**B. Квиз**
```json
{
  "type": "quiz",
  "min_score": 70
}
```

**C. Анкета**
```json
{
  "type": "form",
  "required_fields": ["name", "email"]
}
```

**Процесс проверки:**
1. Получить настройки партнера
2. Проверить каждое условие
3. Если все выполнены - доступ открыт
4. Если нет - показать pending условия
5. Сохранить в `partner_access_log`

#### 4. Кастомизация Партнерами

**Настройки партнера (таблица `partner_referral_settings`):**
```sql
- bot_name              -- Название бота
- bot_avatar_emoji      -- Эмодзи аватара
- welcome_message       -- Приветственное сообщение
- access_conditions     -- Условия доступа (JSON)
- custom_buttons        -- Кастомные кнопки (JSON)
```

**Пример:**
```json
{
  "bot_name": "Моя Академия",
  "bot_avatar_emoji": "🎓",
  "welcome_message": "Добро пожаловать!",
  "custom_buttons": [
    {
      "text": "Наш канал",
      "url": "https://t.me/my_channel"
    }
  ]
}
```

#### 5. Перенаправление в Основного Бота

**После выполнения условий:**
```
Реферальный бот → Основной бот
https://t.me/fel12x_bot?start=ref_123456
```

**Основной бот:**
- Получает пользователя с реферальным кодом
- Связывает с партнером в `user_referrals`
- Начисляет бонусы при покупке

### Команды Реферального Бота

| Команда | Описание |
|---------|----------|
| `/start ref_XXX` | Обработка реферального перехода |
| Callback: `check_sub_XXX` | Проверка подписки |
| Callback: `start_quiz_XXX` | Запуск квиза |
| Callback: `start_form_XXX` | Запуск анкеты |

### База Данных (Реферальный Бот)

**Таблицы:**
```sql
referral_clicks              -- Клики по реферальным ссылкам
partner_referral_settings    -- Настройки партнеров
partner_access_log           -- Лог выполнения условий
user_referrals               -- Связи пользователь-партнер
```

```
1. Партнер создает ссылку
   └─> https://t.me/ref_bot?start=ref_123456

2. Пользователь переходит
   └─> Реферальный бот получает /start

3. Реферальный бот

4. Основной бот
   ├─> Получает /start ref_123456
   ├─> Связывает пользователя с партнером
   ├─> Показывает специальное приветствие
   └─> Начисляет бонусы при покупке

5. При покупке курса
   ├─> Основной бот обрабатывает платеж
   ├─> Начисляет 20% партнеру
   └─> Сохраняет в partner_earnings
```
