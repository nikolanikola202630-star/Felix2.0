## 🚀 За 5 Минут до Production

### Шаг 1: Исправить Безопасность (КРИТИЧНО!)
```bash
node scripts/fix-security.js
```

Этот скрипт:
- ✅ Удалит тестовые файлы с токенами
- ✅ Переместит локальные боты в `local-testing/`
- ✅ Уберет fallback токены из кода
- ✅ Обновит `.gitignore`

### Шаг 2: Проверить Готовность
```bash
node scripts/prepare-deploy.js
```

Проверит:
- ✅ Файловую структуру
- ✅ Безопасность
- ✅ Зависимости
- ✅ Конфигурацию

### Шаг 3: Установить Переменные в Vercel
```bash
# Через CLI
vercel env add BOT_TOKEN
vercel env add REFERRAL_BOT_TOKEN
vercel env add GROQ_API_KEY
vercel env add DATABASE_URL
vercel env add ADMIN_IDS

# Или через Dashboard
# https://vercel.com/dashboard → Settings → Environment Variables
```

**Значения:**
```env
BOT_TOKEN=your_main_bot_token
REFERRAL_BOT_TOKEN=your_referral_bot_token
GROQ_API_KEY=your_groq_key
DATABASE_URL=postgresql://...
ADMIN_IDS=1907288209,8264612178
```

### Шаг 4: Применить Миграции БД

В Supabase SQL Editor выполнить по порядку:
```sql
-- 1. database/migrations/001-add-ml-tables-safe.sql
-- 2. database/migrations/002-academy-tables.sql
-- 3. database/migrations/003-partner-courses.sql
-- 4. database/migrations/004-referral-system-v2.sql
-- 5. database/migrations/005-community-system.sql
-- 6. database/migrations/006-partner-referral-customization-simple.sql
-- 7. database/migrations/007-ai-chat-folders.sql
```

### Шаг 5: Деплой
```bash
# Коммит
git add .
git commit -m "feat: production ready v10.3"

# Пуш (Vercel автоматически задеплоит)
git push origin main
```

### Шаг 6: Установить Webhook'и
```bash
node scripts/sync-bots.js
```

Выбрать `y` когда спросит про установку webhook'ов.

## 🆘 Проблемы?

### Бот не отвечает
```bash
# Проверить webhook
curl https://api.telegram.org/bot<TOKEN>/getWebhookInfo

# Переустановить
node scripts/sync-bots.js
```

### БД не подключается
```bash
# Проверить
node scripts/check-database.js

# Проверить переменные
vercel env ls
```

### AI не работает
```bash
# Проверить логи
vercel logs

# Проверить GROQ_API_KEY
vercel env ls | grep GROQ
```
