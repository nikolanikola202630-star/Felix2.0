## 🎯 Два Способа Применения Миграций

### Способ 1: Автоматический (Рекомендуется)

Используйте скрипт для автоматического применения всех миграций:

```bash
node scripts/apply-migrations.js
```

**Что делает скрипт:**
1. ✅ Проверяет подключение к БД
2. ✅ Создает таблицу `migrations` для отслеживания
3. ✅ Проверяет какие миграции уже применены
4. ✅ Применяет только новые миграции
5. ✅ Использует транзакции (откат при ошибке)
6. ✅ Показывает список всех таблиц

**Требования:**
- `.env.local` с `DATABASE_URL`
- Установленный пакет `pg`

## 🔍 Проверка После Применения

### Проверка 1: Количество Таблиц
```sql
SELECT COUNT(*) as table_count
FROM information_schema.tables 
WHERE table_schema = 'public';
```
Ожидается: **22+** таблиц

### Проверка 2: Индексы
```sql
SELECT 
  tablename,
  indexname
FROM pg_indexes
WHERE schemaname = 'public'
ORDER BY tablename, indexname;
```
Ожидается: **15+** индексов

### Проверка 3: Foreign Keys
```sql
SELECT
  tc.table_name,
  kcu.column_name,
  ccu.table_name AS foreign_table_name,
  ccu.column_name AS foreign_column_name
FROM information_schema.table_constraints AS tc
JOIN information_schema.key_column_usage AS kcu
  ON tc.constraint_name = kcu.constraint_name
JOIN information_schema.constraint_column_usage AS ccu
  ON ccu.constraint_name = tc.constraint_name
WHERE tc.constraint_type = 'FOREIGN KEY'
ORDER BY tc.table_name;
```

## 🆘 Troubleshooting

### Ошибка: "relation already exists"

**Причина:** Таблица уже существует

**Решение:**
1. Пропустить эту миграцию
2. Или удалить таблицу: `DROP TABLE IF EXISTS table_name CASCADE;`

### Ошибка: "column already exists"

**Причина:** Колонка уже существует

**Решение:**
1. Пропустить эту часть миграции
2. Или использовать: `ALTER TABLE ... ADD COLUMN IF NOT EXISTS ...`

### Ошибка: "permission denied"

**Причина:** Недостаточно прав

**Решение:**
1. Проверить что используете правильный DATABASE_URL
2. Проверить права пользователя в Supabase

### Ошибка: "syntax error"

**Причина:** Ошибка в SQL

**Решение:**
1. Проверить что скопировали весь SQL
2. Проверить что нет лишних символов
3. Выполнять по частям
