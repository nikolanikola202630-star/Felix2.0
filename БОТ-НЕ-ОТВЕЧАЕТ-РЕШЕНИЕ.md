## 🔧 РЕШЕНИЯ ПО ПРОБЛЕМАМ

### Проблема 1: Webhook не установлен

**Симптомы:**
- Бот не отвечает
- `getWebhookInfo` показывает пустой URL

**Решение:**

```bash
# 1. Убедитесь, что приложение задеплоено
vercel --prod

# 2. Установите webhook
npm run webhook:set

# Или вручную
curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook" \
  -d "url=https://your-app.vercel.app/api/webhook"
```

---

### Проблема 2: Ошибки доставки webhook

**Симптомы:**
- `last_error_message` в `getWebhookInfo`
- Бот отвечает с задержкой или не отвечает

**Решение:**

```bash
# 1. Проверьте логи Vercel
vercel logs

# 2. Проверьте endpoint
curl https://your-app.vercel.app/api/webhook

# 3. Переустановите webhook
npm run webhook:set
```

**Частые ошибки:**
- `Wrong response from the webhook: 500` - ошибка на сервере
- `Connection timed out` - сервер не отвечает
- `SSL error` - проблема с сертификатом

---

### Проблема 3: Неверный токен

**Симптомы:**
- `Unauthorized` ошибка
- Бот не найден

**Решение:**

```bash
# 1. Проверьте токен в .env.local
cat .env.local | grep TELEGRAM_BOT_TOKEN

# 2. Получите новый токен от @BotFather
# /token в @BotFather

# 3. Обновите в Vercel
vercel env add TELEGRAM_BOT_TOKEN

# 4. Redeploy
vercel --prod
```

---

### Проблема 4: Переменные окружения не установлены

**Симптомы:**
- Ошибки в логах о missing env variables
- 500 Internal Server Error

**Решение:**

```bash
# Проверьте переменные в Vercel
vercel env ls

# Добавьте недостающие
vercel env add TELEGRAM_BOT_TOKEN
vercel env add SUPABASE_URL
vercel env add SUPABASE_KEY
vercel env add GROQ_API_KEY

# Redeploy
vercel --prod
```

---

### Проблема 5: API endpoint не работает

**Симптомы:**
- 404 Not Found
- Webhook получает ошибки

**Решение:**

```bash
# 1. Проверьте vercel.json
cat vercel.json

# 2. Проверьте api/_router.js
cat api/_router.js

# 3. Проверьте endpoint вручную
curl https://your-app.vercel.app/api/webhook

# 4. Redeploy
vercel --prod
```

## 🔍 Детальная диагностика

### 1. Проверить токен бота

```bash
curl "https://api.telegram.org/bot<YOUR_TOKEN>/getMe"
```

Должно вернуть:
```json
{
  "ok": true,
  "result": {
    "id": 123456789,
    "is_bot": true,
    "first_name": "Your Bot",
    "username": "your_bot"
  }
}
```

### 2. Проверить webhook

```bash
curl "https://api.telegram.org/bot<YOUR_TOKEN>/getWebhookInfo"
```

Должно вернуть:
```json
{
  "ok": true,
  "result": {
    "url": "https://your-app.vercel.app/api/webhook",
    "has_custom_certificate": false,
    "pending_update_count": 0
  }
}
```

### 3. Проверить endpoint

```bash
curl https://your-app.vercel.app/api/webhook
```

Должно вернуть что-то (не 404).

### 4. Проверить логи

```bash
vercel logs --follow
```

Отправьте сообщение боту и смотрите логи.

